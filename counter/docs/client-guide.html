架构概览

laser-mock-counter 是一个 Spring Boot 客户端，通过 Aeron Cluster Client API 向 laser-matching-engine Aeron 集群发送 SBE 编码的交易指令（下单 / 撤单 / 改单）。 它对外提供 REST API，支持单笔和批量订单操作。

AeronClusterClientService 设计

AeronClusterClientService 是客户端与 Aeron 集群通信的核心组件。 采用 MediaDriver.launch() + 固定目录 + 独立守护线程模式，确保高可用和资源安全。

核心设计要点
设计说明
MediaDriver.launch() + 固定目录 MediaDriver 使用固定目录 /tmp/laser-client/aeron，SHARED 线程模型， dirDeleteOnStart=true + dirDeleteOnShutdown=true。 启动一次永久复用，不随连接重建而销毁。
守护线程初始连接 初始连接在独立守护线程 aeron-client-connect 上进行，不阻塞 Spring 启动。 连接失败时不影响应用启动，待首次 offer() 调用时自动重连。
独立心跳线程 使用 ScheduledExecutorService 每 3 秒执行 sendKeepAlive() + pollEgress()， 维持 Aeron 会话活跃（服务端 10 秒超时）。心跳失败时自动标记断连状态。
懒重连模式 心跳检测到断连后标记 isConnected=false，不立即重连。 下次 offer() 调用时通过 ensureConnected() 触发重连，避免空转。
线程安全 ReentrantLock 防止多线程并发重连；AtomicBoolean / AtomicInteger 管理状态； volatile 修饰 aeronCluster 保证多线程可见性。
背压重试 offer() 发送消息时，遇到背压会重试最多 10 次， 每次重试间调用 pollEgress() + SleepingIdleStrategy。
生命周期

断线重连机制

AeronClusterClientService 内置了自动断线重连机制。当与集群的连接断开时（如集群重启、网络中断）， 客户端会自动检测并以指数退避策略持续重试，直到重新连接成功或达到最大重试次数。

工作原理
stateDiagram-v2 [*] --> 守护线程连接: @PostConstruct 守护线程连接 --> 已连接: 连接成功 守护线程连接 --> 等待首次请求: 连接失败 已连接 --> 心跳保活: 每 3 秒 sendKeepAlive 心跳保活 --> 心跳保活: 正常 心跳保活 --> 标记断连: 异常 标记断连 --> 等待offer触发: isConnected=false 等待offer触发 --> 指数退避重连: offer() → ensureConnected() 等待首次请求 --> 指数退避重连: offer() → ensureConnected() 指数退避重连 --> 已连接: 重连成功 指数退避重连 --> 重连失败: 达到最大次数 已连接 --> [*]: @PreDestroy
重连参数
参数值说明
MAX_RECONNECT_ATTEMPTS 10 单次重连流程的最大尝试次数
INITIAL_RECONNECT_DELAY_MS 1,000 首次重连等待时间（1 秒）
MAX_RECONNECT_DELAY_MS 30,000 最大重连等待时间（30 秒）
退避乘数 2 每次失败后等待时间翻倍
CONSECUTIVE_FAILURE_THRESHOLD 2 连续失败次数达到此阈值后触发重连
HEARTBEAT_INTERVAL_SECONDS 3 心跳间隔（秒），需小于服务端 10 秒会话超时
退避策略示例

第 1 次1 秒
第 2 次2 秒

第 4 次8 秒
第 5 次16 秒
第 6 次及以后30 秒（上限）
核心实现
// AeronClusterClientService.java — 指数退避重连
private void reconnect() {
    reconnectLock.lock();
    try {
        if (isConnected.get()) return;  // 双重检查

        // 关闭旧连接
        if (aeronCluster != null) aeronCluster.close();

        long delay = INITIAL_RECONNECT_DELAY_MS;  // 1秒
        for (int attempt = 1; attempt <= MAX_RECONNECT_ATTEMPTS; attempt++) {
            try {
                aeronCluster = connectToCluster();  // 复用 MediaDriver
                return;  // 连接成功
            } catch (Exception e) {
                log.warn("Reconnection attempt {}/{} failed", attempt, MAX_RECONNECT_ATTEMPTS);
                if (attempt < MAX_RECONNECT_ATTEMPTS) {
                    Thread.sleep(delay);
                    delay = Math.min(delay * 2, MAX_RECONNECT_DELAY_MS);  // 翻倍，上限 30s
                }
            }
        }
        log.error("Failed to reconnect after {} attempts", MAX_RECONNECT_ATTEMPTS);
    } finally {
        reconnectLock.unlock();
    }
}
关键设计
  • MediaDriver 不销毁：重连时只关闭旧的 AeronCluster，MediaDriver 始终保持运行，避免资源重建开销。
  • ReentrantLock 防并发：多个线程同时触发重连时，只有一个线程执行，其余等待后通过双重检查直接返回。
  • 懒触发：心跳失败仅标记 isConnected=false，实际重连由 offer() 触发，避免无业务时空转重连。
  • 连续失败计数：offer() 背压失败和心跳失败均计入 consecutiveFailures，达到阈值 2 后也会触发重连。

SBE 编码

所有订单消息在通过 Aeron Cluster ingress 通道发送前，均使用 Simple Binary Encoding (SBE) 进行编码。SBE 提供零拷贝、无内存分配的序列化方式，非常适合低延迟交易系统。

graph LR REQ["PlaceOrderRequest
(Java POJO)"] ENC["request.encode(buffer, 0)
SBE 编码器"] BUF["MutableDirectBuffer
byte[]"] OFFER["cluster.offer(buffer, 0, len)"] UDP["Aeron UDP Ingress"] REQ --> ENC --> BUF --> OFFER --> UDP
