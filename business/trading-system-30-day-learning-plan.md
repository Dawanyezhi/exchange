# 交易系统学习计划 v2

## 这版计划的定位

这是一份面向工程视角的 `30 天交易系统学习计划`。

目标不是背概念，而是用 30 天建立一套相对完整的认知框架：

- 看懂交易系统从 `下单 -> 风控 -> 订单管理 -> 撮合/执行 -> 行情 -> 账务 -> 清算/结算 -> 监控恢复` 的完整链路
- 区分 `接入层 / 会话层 / OMS / 风控 / 撮合 / 行情 / 账户与持仓 / 台账与结算 / 回测与仿真 / 运维与可观测性` 的职责边界
- 理解交易系统中的关键工程问题：`顺序性`、`幂等性`、`状态机`、`回放恢复`、`低延迟`、`一致性`
- 最终能够画出一个最小可用交易系统的设计图，并解释每个模块为什么存在

## 本版明确不包含

本版 **不学习国内期货业务规则**，因此有意识去掉以下内容：

- 国内期货交易所规则细节
- `CTP` 生态和柜台接入细节
- 开平、今昨、强平、夜盘等国内期货特有规则
- 国内期货保证金与结算细则

如果后续你想补这部分，建议单独开一个“国内期货业务规则专题”，不要和通用交易系统架构学习混在一起。

## 30 天后希望达到的能力

- 能讲清楚一条订单从客户端发起到最终成交、记账、对账的全过程
- 能解释 `OMS`、风控、撮合、行情、账务、结算之间的数据流和控制流
- 能看懂主流交易协议文档中的关键字段、消息类型和会话语义
- 能解释为什么交易系统经常要围绕 `状态机`、`序号`、`快照+增量`、`日志回放` 设计
- 能输出一版自己的“简化交易系统 MVP 设计图”

## 学习方法

建议每天投入 `90-120 分钟`，固定做 5 件事：

- 读资料 `35-45 分钟`
- 画图或列状态机 `20 分钟`
- 摘术语和字段 `10-15 分钟`
- 做一个小练习 `15-20 分钟`
- 写总结 `10-20 分钟`

每天都要有一个小产出，不要只停留在“看过了”。

## 先建立模块地图

在正式开始前，先记住下面这 10 个模块：

1. `Client / API Gateway`
2. `Session / Protocol`
3. `OMS`
4. `Pre-trade Risk`
5. `Matching / Execution`
6. `Market Data`
7. `Account / Position`
8. `Ledger / Clearing / Settlement`
9. `Simulation / Backtest / Paper Trading`
10. `Observability / Recovery / Operations`

后面的 30 天，基本就是围绕这 10 个模块打通认知。

## 阅读时要始终带着的 6 个问题

- 这个模块解决的核心问题是什么
- 它的输入、输出、状态分别是什么
- 它依赖什么上游，又给谁输出结果
- 哪些规则是业务规则，哪些是系统约束
- 它最怕什么故障，如何恢复
- 线上如果出问题，最先看什么日志、指标、序号或状态

## 30 天学习计划

## Phase 1：先建立全景与订单生命周期

### Day 1：建立交易系统全景图

- 阅读资料：Coinbase Trading Concepts
- 学习重点：交易系统里都有哪些基础对象，例如 `product`、`order`、`trade`、`account`
- 输出内容：一张“交易系统全景图”
- 小练习：用自己的话解释 `order`、`fill`、`position`、`balance`
- 复盘问题：为什么一个交易系统不等于一个撮合引擎

### Day 2：区分参与者与职责边界

- 阅读资料：Coinbase Exchange Trading Concepts
- 学习重点：用户、账户、组合、订单、成交之间的关系
- 输出内容：一张“用户-账户-订单-成交关系图”
- 小练习：列出账户系统和撮合系统各自维护的核心状态
- 复盘问题：为什么账户体系不能直接嵌进撮合模块里

### Day 3：认识系统分层

- 阅读资料：算法交易系统架构，此篇足矣！
- 学习重点：前台、接入、OMS、风控、执行、行情、账务、清算的系统分层
- 输出内容：一张“交易系统分层图”
- 小练习：把每层最核心的 2 个职责写出来
- 复盘问题：哪些层偏业务，哪些层偏工程基础设施

### Day 4：理解订单生命周期

- 阅读资料：Coinbase Exchange Matching Engine

### Day 5：理解订单类型与时间有效性

### Day 6：理解 OMS 的存在价值

- 输出内容：一张“OMS 职责边界表”
- 小练习：列出至少 8 个应该由 OMS 负责的能力
- 复盘问题：什么逻辑放在 OMS 更合理，什么逻辑应该下沉到别的模块

### Day 7：建立订单主链路时序图

- 阅读资料：回顾 Day 1-Day 6
- 学习重点：报单、校验、路由、成交、回报、持仓更新、资金更新
- 输出内容：一张“订单主链路时序图”

- 复盘问题：哪些步骤必须严格顺序执行，哪些可以异步化
